java 实战之路

  1. 入手建议

    • 视频+看书入门：可以先看视频，更容易进入，之后针对不了解的地方看书。看不进去书可以采取抄书的方法，写在博客上之类的。

    • 可以先实现一个最基础的业务：java仓库增删改查，可以先看看别人的项目怎么做，注重业务逻辑的建立，理清楚需要哪几个业务模块

    • 不一定要自己动手造轮子，可以先找找别人的代码，关键是理解整合优化

    • 项目最好贴合自己的需求，更有动力，开始时最好找那种可以参考的目标项目，分析他现有的功能，进行复现和优化

  2. 工作建议

    • 及时沟通进度，让boss看到你的工作量和进度，不要害怕问问题

    • 提高自己实现想法的能力，善于利用平台

    • 在没有代码或者也没有业务输出时，可以写一写文档，也是一种知识输出

    • 将工作和个人分开，不要玻璃心

  3. 推荐的阅读项目：

    • hutool（java工具集）

      注释十分清楚，结构也很清晰

    • PiggyMetrics（springboot, springcloud）

      多模块，较为复杂

    • HanLP

      NLP自然语言处理，例子和算法解释得十分清楚

  4. 如何搞懂一个开源项目

    • 项目是干什么的，有哪些技术点
    • 在本地将项目运行起来
    • 调试弄懂功能点
    • 自己做出一些修改
以下为随意记的笔记：
1. 认识源代码

首先java一般有两个下项目框架： maven/gradle

以gradle为例，打开src->build.gradle记录所有依赖的包文件

而maven是记录在pom.xml中

通过这个文件的阅读可以知道，这个项目用了哪些技术点

2. 项目结构：只看src下面的 (以Halo项目为例子)
src->main

  • **->java **（Java源代码）

    第一步运行！ springboots项目入口在->run.halo.app->utils->Application

    • 可以从controller开始看起：控制器，它记录了前后端交互所调用的接口

    可以通过点击函数名，跳转到具体的函数位置，如果是接口，点击行前左侧向下的箭头跳转至实现的函数位置，再点击左上方小轮子 ，目录也会定位到这个函数当前位置

    • 一般会到service，它是为contoller进行服务，实现具体业务
    • repository数据库相关操作
    • model记录一系列封装对象

    (不是主要需要看的)

    util工具类

    security登录安全认证

  • **->resources **（资源文件）

    templates里面

    • 配置文件 .yaml
    • banner.txt 设置命令行启动效果,springboots可以自动识别
    • ->themes.anatole/common->source .ftl 静态页面文件，包括侧边栏等等

    admin/static.js/templates里面都是与前端相关的CSS等等文件

PS：建议从springboots开始入手学，不用先从很底端开始
3. 调试

右上方有一个小虫子，点击开始调试模式

打开网页的应用，按下F12，打开调试界面

视频后半段有完整的调试解说，有需求的时候跟着搞一遍就可以
