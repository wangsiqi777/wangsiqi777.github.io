1. 面向对象

public 所有

protected 包内+子类

默认 包内

private 本类

2. 封装

使用get/set方法设置和访问类内private变量

对象在堆内存内开辟了一块空间

系统默认提供构造方法，默认是无参的；但如果实体类中有显式的构造方法，系统就不会再自动提供无参的方法

重载：一个类中多个同名方法，且参数列表不一致

3. 继承 extend

子类会继承父类所有非私类方法（父亲把私房钱藏起来了，儿子继承不了）

重写：方法名一样，返回类型一样，方法修饰权限基本一样，参数列表必须一样，捕获异常基本一样

多态：前提是子类中有父类的重写。编译的时候处理父类，执行的时候处理子类

4. 抽象

没有方法体的方法，抽象方法

抽象方法必须由abstract修饰

包含抽象方法必须是抽象类

抽象类不能被new（直接new，会调用没有实体的方法）

抽象类天生就能被继承

需要子类去继承父类，去重写@Override抽象方法实现，其他类new子类

5. 接口

接收参数，有返回，但不管具体实现–>抽象方法的集合

一个抽象方法一个接口：

同理 main在调用的时候，也是只能继承实现了接口的接口子类

按下滚轮，列模式编辑

接口所有方法都是public abstract

接口所有成员变量都是常量public static final

EG:

 public abstract void kele();

异常

如何处理异常，让他不影响程序的主流程

  1. 运行时
  2. 编译时
try{}
catch(异常类型){}
finally{
//天荒地老海枯石烂都会运行
}

按照家族树的顺序安排异常

finally执行位置：

建议多使用try-catch，

若在一个语句声明后后面写throws，表明有可能出异常（此时try-catch捕获不到异常）

写了它，就不会报实体里面的编译时异常

声明的异常找到了主方法，程序终止
