新闻可视化——四川大学新生数据可视化实战

2020年5月14日
| 数据可视化

前言

两会期间 Who Runs China 爆热无疑传达出新闻数据可视化已经成为数据可视化重要分支的讯号。用准确生动的图表揭露大数据的秘密是老百姓们所喜闻乐见的。

为了宣传可视化社团，我向阿婷提议可以借用我在大川公众号的编辑身份，推出一期可视化推送。但由于缺乏可靠的数据源，迟迟没有定下主题。最后，我们锁定了每一年的新生数据分析。对比川大和其他高校的去年新生推送可以发现，它们普遍采用的都是手绘制图的形式，图表也趋向于基础的折线和柱图。不具有较强的新颖性，数据的刻画也不够精确。基于此，确定下了本次推送制图的目标：

  • 多用动图展示交互，增加阅读兴趣

  • 少用条形图和折线图，尽可能展示不同类型图表的表达魅力

最早萌生这个想法时，Echarts还不支持微信直接嵌入代码，我们只能录gif 和截图，舍弃部分交互来做数据呈现，但是效果还是很棒，在看到一位家长用我做的图作为自己的微信头像时，那种满足感是无与伦比的。

目前Echarts已经支持微信直接嵌入，我也十分看好数据新闻未来的前景，它能够生动反映手绘所不能带来的惊喜和精确。非常遗憾的是原稿由于U盘的损坏，代码文件已经不可找回了，只能用渣像素来讲解一下自己做的时候用到的方法和思考。

沉痛提醒大家以后在做项目时，要做好版本迭代的管理和备份工作。还有就是不要偷懒，长时间将U盘插入电脑不拔出Orz

推送链接：姓氏秘密、星座比拼、男女比例……来看川大本科新生大数据！

制图分析

1. 散点地图

第一幅图分析的是新生家乡分布，选择了表意直观的地图，通过添加关联边的动态效果可以很好地吸引读者继续阅读的兴趣，也有切合“海纳百川”的校训之意。通过比较圆形的大小就可以直观感受各省生源分布情况。

出现的问题是由于四川本地的生源数量过大，四川地区的圆点会与重庆等地区的圆点发生非常严重的堆叠现象。所以直接使用了红色的箭头标识其为目的地，将这幅图的含义改为反映外省生源的分布，但在鼠标悬停时，可以看到每个省份具体生源的数量。

其次，就是制作一篇新闻推送所需要的严谨性。因为地图太小了，肉眼观察不能保证钓鱼岛一定在图上，我只好按照大概的经纬位置，手工加上了一个小点和名字。不过功夫不负有心人，后来我在评论区看到一位家长使用了这幅图作为头像，真的是超级有成就感~

2. 性别-年龄-学院 桑基图

从桑基图中能够很清楚地看到不同年龄占男女生总人数的比例，以及占学院总人数的比例。最小的同学仅有14岁，来自化学学院，而年龄最长的同学是37岁的学长，来自经济学院。厉害了，我的川！

在讨论性别-年龄-学院关系时，主要考虑了两种：多维平行坐标图和桑基图。

多维平行坐标图的交互是采用画框选中的方式，在这种分支太多时，很难精准框选展示每一个年龄段的相关连线。而桑基图能通过块及其相关边块高亮清晰的看出每一个年龄段下性别和学院的分布，所以选择了桑基图。

导入数据后，又出现了新的问题：年龄最小/大的人数过少，根本无法选中图表中对应的年龄块使其相关连线高亮。学院又过多，直接按比例放大，手机屏是无法承载的。就在快要放弃动图演示的时候，偶然发现通过增大边框宽度，即将 series->data->itemStyle->borderWidth 中的数值改大，就可以做到正常交互，又适应屏幕宽度。问题得到成功解决~（现在还记得那天肝图到凌晨给阿婷发消息说有办法，在电脑前傻笑的样子hhh）

比较可惜的是，实际在手机端查看时候，gif 像素被压缩了，并不能很清晰看到各个区块的名字。不过现在微信已经支持代码的直接嵌入，应该不会再有这个问题了。

3. 星座矢量图 / 民族图

为了规避掉常规的分类柱状图，选取了圆形环状图，圆的大小对应了人数，环状布局也可以让人联想到星盘。但是光秃秃的圆形显然还是不够美观，为了进一步增加图形的趣味，又从阿里icon图库上挑选了星座矢量图与圆形环状图做叠加显示。

民族分布图的寓意取自于56个民族56朵花。图表的原理是将基础的饼图用两层结构改成环图，然后中间设置固定间隔，按比例分割环图，就形成了花朵的形状。

4. 文字云

文字云是可视化进入大众视线的一个重要作品。我们选定了最能代表四川的两个物象——熊猫和竹子，分别对新生的姓和名中的字频率制作了词云。这一部分的绘图是由阿婷完成的，主要使用的是python的wordcloud库。

推荐教程：python WordCloud 实现词云

#导入wordcloud模块和matplotlib模块
from wordcloud import WordCloud,ImageColorGenerator,STOPWORDS
import matplotlib.pyplot as plt
from scipy.misc import imread
import chardet

#读入处理过的姓名文件
text2= open("name7.txt","r",encoding="UTF-8").read()
#读入背景图片
pic = imread("panda16.png")
#生成词云
cloud = WordCloud(mask=pic,prefer_horizontal=1,#width=800,height=800,
                  font_path="C:\\Windows\\Fonts\\Nk728iWCZ.TTF",
                  background_color=None,scale=5,
                  max_font_size = 100,mode="RGBA") 
cloud.generate(text2)
image_colors = ImageColorGenerator(pic)
cloud.recolor(color_func=image_colors)
#显示词云图片
plt.imshow(cloud)
plt.axis("off")
plt.show()
#保存图片
cloud.to_file("test26.png")

WordCloud函数参数详解

mask：设置为pic,让生成的词云具有图片的形状。#FFFFFF是不会显示字的，因此如果画面背景不是纯白也会显示字。
prefer_horizontal： 这个属性是让竖直的字的出现概率为1，因此词云的所有字都是竖直排列的，可读性会更强。
background_color=None和mode=’RGBA’ ：这两个一起设置，能够让图片的背景是透明的。（这里主要是需要和美工交接，让她上底色）

结语

在这次新生数据推送中，我在视图选择上很刻意地规避了常规图表，努力营造酷炫的感觉。但在更日常的新闻可视化中，如何让读者更清晰快速的了解数据背后的含义才是最重要的。折线和柱状有着天然的简明易懂的优点，是值得被好好运用的。

在这次疫情中，新闻可视化大放异彩，简单清晰的疫情地图牵动着每个国人的心，也精确刻画了战疫攻坚的每一步。期间也有幸加入北大可视化团队的疫情可视化文案组搬了两周砖。（不过很遗憾由于某些原因，这个项目最终暂停了）

一图胜千言，相信未来会有更多的优秀作品，也期待未来有更多机会探索和展示这些有趣的数据~
