搭建hugo平台网站（小白指南）

1.首先尝试的是hexo，页面主题友好，可选择范围广

推荐搭建链接：

2.配置到最后一步，偶然查阅资料发现，hexo在文章数超过200时，会非常缓慢，不适合我这种废话大王。因此选择更换为hugo进行博客搭建

优点：文章数较多的时候，访问速度快；部署到github上的方法相较于hexo，更为简单

缺点：配置教程较少

3.过程中的学习经验

生成博客：

使用命令hugo new post/file_name.md，Markdown文件将生成在二级目录post下面

打开新建Markdown文件之后，文章开头会有默认的被+++包住的内容，这些内容即是该文章的配置。在/mysite/archetypes/default.md中可修改默认的配置。

+++
title = "{{ replace .Name "-" " " | title }}"  # 文章标题
date = {{ .Date }}  # 自动添加日期信息
draft = true  # 设为false可被编译为HTML所有人可见，true仅在本地查看修改
tags = [""]  # 文章标签，可设置多个，用逗号隔开。Hugo会自动生成标签的子URL
comments = true  # 是否开启Disqus评论功能
share = true  # 是否开启分享
+++

使用 hugo server 启动服务，生成或更新public文件夹，可以在本地通http://localhost:1313/ 观察页面变化

建议将本地仓库/ShaneTian.github.io建在在/mysite外的平行的文件夹中，每次更新文章后只用将生成的/public文件夹的静态网站内容复制到/ShaneTian.github.io，然后再push到远程仓库即可。
